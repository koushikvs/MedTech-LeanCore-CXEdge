Deployment & Runtime Architecture

Decoupled front-end and back-end, both running in containers within the client AWS VPC. Identity via OKTA, integration via MuleSoft, SAP CCv2 as the core commerce engine, IoT ingestion via MQTT, and a Bitbucket → Veracode → ECR CI/CD pipeline.

User Entry · Edge

Customer browser → DNS → CDN → ALB

All traffic enters through a CloudFront-fronted aliased domain. TLS termination at the edge, cert managed automatically.

TLS · cert

CloudFront Distribution
CDN · edge

Domain aliased CDN. Registered origins differentiate FE vs. BE behaviors via path.

Client AWS VPC

Application layer — Front End & Back End containers

Decoupled FE and BE, both behind their own ALBs, deployed on Fargate ECS. The decoupling principle is the architectural cornerstone — front-end can iterate independently of the back-end and core systems.

Front End ALB → Containers

Angular Docker Nginx

Angular SPA served via Nginx, containerized. The FE container is the only thing the customer's browser actually loads. ECS logs flow to CloudWatch.

Back End ALB → Containers

Spring Boot Docker Java

Spring Boot REST API in Java, containerized. Calls into MuleSoft for SAP & data-lake integration; reads/writes its own data tier.

↓ Identity, integration & SAP ↓
Identity Layer

Okta CIAM/WIAM — JWT auth across edge applications

A shared identity layer powers self-registration, role-based access and JWT-validated API calls. BE containers introspect tokens against Okta on every privileged call.

Okta
CIAM / WIAM

Customer Identity (CIAM) for clinical users + Workforce Identity (WIAM) for internal admins. JWT introspection on protected APIs.

Integration · SAP & Data

MuleSoft API Gateway — bridging the edge to SAP S/4HANA & Data Lake

Every call from the BE containers to SAP or to the data lake flows through MuleSoft — a single integration layer that protects the SAP core from edge volatility.

Why this shape

🛡️

Lean Core protected

SAP S/4HANA & CCV2 are reached only through MuleSoft. The edge cannot bypass governance — and the core is shielded from edge volatility.

⚡

Edge moves at edge speed

FE and BE deploy independently via Bitbucket → Veracode → ECR → Fargate. Two-week speed-to-market wasn't an accident — it was an architectural choice.

🔐

Identity & secrets centralized

Okta is the single auth surface. Secrets Manager rotates credentials. Veracode gates code. Each control lives in one place — easier to audit, easier to harden.

Why this shape

🧱

Front-end is composable

Storybook design system + atomic components mean every market and every app share the same primitives. New experiences come together fast — and look consistent.

🪢

BFF absorbs complexity

Each consumer-facing experience gets a tailored backend that orchestrates downstream calls. Front-ends stay thin; integration logic stays in one disciplined layer.

🏛️

Core stays clean

SAP CCV2 + S/4HANA + the broader enterprise systems sit behind the ESB. They're invoked, not modified — exactly the Lean Core principle in code.
